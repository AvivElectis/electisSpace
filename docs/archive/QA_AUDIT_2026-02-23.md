## 🔴 Critical / High Severity — FIXED

### 1. XSS via `dangerouslySetInnerHTML` (High — Security)
**File:** `src/features/update/presentation/UpdateDialog.tsx:123`  
**Issue:** GitHub release notes rendered with `dangerouslySetInnerHTML` without sanitization. A compromised or malicious GitHub release could inject scripts.  
**Fix:** Added `DOMPurify.sanitize()` wrapper. Installed `dompurify` package.

### 2. Incorrect authorization middleware on admin route (High — Security)
**File:** `server/src/features/auth/routes.ts:124`  
**Issue:** `authorize('PLATFORM_ADMIN')` uses the store-role checker. While it works because `authorize()` has a PLATFORM_ADMIN global-role bypass, the intent is wrong—`PLATFORM_ADMIN` is not a store role. If the bypass logic ever changes, this route would break open or lock out admins.  
**Fix:** Changed to `requireGlobalRole('PLATFORM_ADMIN')` which directly checks `globalRole`.

### 3. Console.log statements leaking debug info in production (High — Code Quality / Security)
**Files:**
- `src/features/settings/presentation/StoreDialog.tsx` — logged store creation payloads
- `src/features/people/infrastructure/peopleService.ts` — logged full article JSON including business data
- `src/shared/infrastructure/services/storeEventsService.ts` — logged SSE events and tokens

**Fix:** Removed production console.logs; kept dev-only `console.debug` behind `isDev` flag where useful.

## 🟡 Medium Severity — Documented

### 4. Refresh token rotation race condition (Medium — Security)
**File:** `server/src/features/auth/service.ts` — `refresh()` method  
**Issue:** The refresh flow reads the stored token, revokes it, and creates a new one in separate queries. Under concurrent requests (e.g., multiple tabs), this can cause token theft detection false positives or allow token reuse in a narrow window.  
**Recommendation:** Wrap the read-revoke-create in a database transaction with row-level locking.

### 5. User cache not invalidated on role/store changes (Medium — Security)
**File:** `server/src/shared/middleware/auth.ts`  
**Issue:** The in-memory user context cache (60s TTL) means role changes take up to 60 seconds to propagate. `invalidateUserCache()` is exported but only called in limited places.  
**Recommendation:** Call `invalidateUserCache(userId)` in all user/store/company update services. Consider Redis-based cache for multi-instance deployments.

### 6. Static salt in encryption key derivation (Medium — Security)
**File:** `server/src/shared/utils/encryption.ts:28`  
**Issue:** Uses a hardcoded salt `'electisspace-encryption-salt-v1'` for PBKDF2 key derivation. While the comment explains this is for deterministic derivation, it reduces security compared to per-record salts.  
**Recommendation:** For new encryption contexts, consider per-record random salts stored alongside ciphertext.

### 7. Missing Helmet `referrerPolicy` hardening (Medium — Security)
**File:** `server/src/app.ts`  
**Issue:** Helmet defaults are used for referrer policy. Consider setting `referrerPolicy: { policy: 'strict-origin-when-cross-origin' }` explicitly.

### 8. No request body size limit on specific routes (Medium — Performance)
**File:** `server/src/app.ts:84` — global `10mb` limit  
**Issue:** The 10MB JSON body limit is generous. Bulk import endpoints may need it, but most endpoints should have tighter limits.  
**Recommendation:** Add per-route body size limits for sensitive endpoints.

### 9. Missing `react-window` for large lists (Medium — Performance)
**Observation:** `react-window` is installed but verify it's used for all potentially large lists (people, spaces). Some list components may render all items.

### 10. Bcrypt compare on every stored refresh token (Medium — Performance)
**File:** `server/src/features/auth/service.ts` — `refresh()` method  
**Issue:** On token refresh, all user's non-revoked tokens are fetched and each is compared with bcrypt. With many sessions, this is O(n) bcrypt operations.  
**Recommendation:** Store a token identifier (JTI) alongside the hash and look up by JTI first.

## 🟢 Low Severity — Documented

### 11. Console.logs in server startup (Low — Code Quality)
**File:** `server/src/server.ts`  
**Issue:** Startup banner and status messages use `console.log`. These are acceptable for startup but could use the structured `appLogger` for consistency.

### 12. Missing ARIA labels on icon-only buttons (Low — Accessibility)
**Observation:** Several icon-only buttons (close, settings) may lack `aria-label`. Requires component-by-component review.

### 13. Missing `<meta name="description">` for PWA (Low — PWA)
**Observation:** Verify the PWA manifest and index.html include proper meta tags for description, theme-color, and apple-touch-icon.

### 14. `crypto-js` still in frontend dependencies (Low — Code Quality)
**File:** `package.json`  
**Issue:** `crypto-js` is listed as a dependency. The server uses Node's native `crypto`. If `crypto-js` is only used for legacy decryption migration on the frontend, consider removing it after migration is complete to reduce bundle size.

### 15. Unused `result` variable (Low — Code Quality)
**File:** `src/features/settings/presentation/StoreDialog.tsx`  
**Issue:** After removing console.log, `const result = await companyService.createStore(...)` — `result` may be unused. Compiler will warn.
